Summary of Attack Chain

  1. Token Theft: Attacker obtains JWT token via XSS, malicious extension, or MITM
  2. CORS Exploitation: Attacker makes cross-origin request to api.immutable.com/v2/rewards/earnings
  3. Data Extraction: API returns sensitive user data due to access-control-allow-origin: *
  4. Exfiltration: Attacker sends stolen data to their server

Impact: Personal user data (earnings, quests, gems) can be stolen by any website.

How to Reproduce with Real Data

  1. Login to play.immutable.com
  2. Open DevTools (F12) → Network tab
  3. Reload page, find a request to api.immutable.com
  4. Click the request → Headers tab → find Authorization: Bearer eyJ...
  5. Copy the token value (without "Bearer ")
  6. Paste token in Step 2 above
  7. Click buttons in sequence to see full attack chain

Security Impact & Remediation

Problem: access-control-allow-origin: * on authenticated endpoints

Fix: Restrict CORS to specific trusted origins only

Required Changes:

# Bad (current)
access-control-allow-origin: *

# Good (fixed)
access-control-allow-origin: https://play.immutable.com
# or implement proper origin validation
